2. Handling System Interruptions During File Comparison

During testing, a possible edge case was identified. While comparing the previous file set (previousset1) with the current file set (currentset1) stored in STL set<string>, a situation may occur where the system reboots or shuts down in between the comparison process (for example, during a command like apt purge google-*).

To prevent data loss in such scenarios, a solution was proposed:

Create a file named previousset1.txt.

Store the sorted list of filenames from the directory in this file.

On program restart (after reboot or interruption), the program can reload this file and reconstruct the previous snapshot.

This approach ensures that the comparison process remains consistent even after reboot, shutdown, or unexpected interruptions.

Using a .txt file for snapshot storage is best because:

.The directory size is relatively small (< 10,000 files)

.The dataset is small

.Low memory usage

Sr. No.	Command as typed	Right command (if different)	Explanation

3	fdisk /dev/sdb	–	Start interactive partition editor for /dev/sdb.
4	(inside fdisk) m	–	Display the help menu inside fdisk.
5	(inside fdisk) p	–	Print the current partition table.
6	(inside fdisk) d (then default partition 4)	–	Delete partition 4 (the Empty partition).
7	(inside fdisk) d (then default partition 2)	–	Try to delete partition 2 – fails because it’s invalid.
8	(inside fdisk) d (then 1)	–	Delete partition 1.
9	(inside fdisk) p	–	Print partition table again – only partition 2 remains (corrupt).
10	(inside fdisk) m	–	Show help again.
11	(inside fdisk) n	–	Create a new partition.
12	(inside fdisk) p (primary)	–	Choose primary partition type.
13	(inside fdisk) accept default partition number 1	–	Use partition number 1.
14	(inside fdisk) accept default first sector 2048	–	Start at the first available sector.
15	(inside fdisk) accept default last sector (whole disk)	–	Use the remaining space for the partition.
16	(inside fdisk) p	–	Print the new partition table (now has /dev/sdb1 and the leftover /dev/sdb2).
17	(inside fdisk) w	–	Write the new partition table to disk and exit.
18	lsblk	–	Verify that the new partition is visible (still no filesystem, just the disk).
20	mkf (then Tab to see completions)	–	Partial command – shows available mkfs variants (not an actual command).
21	mkf (again)	–	Same as above.
22	mkfs.ext /dev/sdb (then Tab)	–	Partial command – Tab shows mkfs.ext2, mkfs.ext3, mkfs.ext4.
23	mkfs.ext4 /dev/sdb	–	Create an ext4 filesystem on the whole disk /dev/sdb (not just a partition).
24	lsblk	–	Confirm that the disk now has a filesystem but is still not mounted.
25	mount /dev/sdb /media/ (then Tab)	–	Partial mount command – Tab shows possible mount points under /media/.
26	mount /dev/sdb /media/sdb1	–	Mount the USB drive (with its new ext4 filesystem) at /media/sdb1.
27	cd /media/sd (then Tab)	–	Partial cd – Tab shows available directories (sdb1, sdb2, …).
28	cd /media/sdb1/	–	Change into the mounted directory.
29	a (alias for ls -la)	–	List all files (including hidden) with details – shows lost+found directory.
30	cd lost+found/	–	Enter the lost+found directory (used by fsck).
31	a	–	List contents of lost+found – it is empty (only . and ..).
32	cd ..	–	Go back to the parent directory (/media/sdb1).
33	a	–	List again – shows the same lost+found directory.
